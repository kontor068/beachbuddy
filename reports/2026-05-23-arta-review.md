# Έλεγχος Άρτας - 2026-05-23

## Απλά λόγια

Έγινε deep audit για `Arta`.

Δεν πρόσθεσα νέες παραλίες στην Άρτα.

Ο λόγος: στο ευρύτερο scan του Αμβρακικού βρέθηκαν 9 OSM beach candidates, αλλά μόνο 2 ανήκουν στην Περιφερειακή Ενότητα Άρτας και είναι ήδη μέσα στο dataset.

## Τρέχον αποτέλεσμα

- Παραλίες Άρτας στο dataset: 2
- BLOCKER: 0
- HIGH: 0
- MEDIUM: 0
- LOW: 0
- Το audit περνάει.

Οι 2 υπάρχουσες εγγραφές που μένουν:

- Παραλία Σαλαώρας
- Φράχτης

## Τι δεν πέρασα στην Άρτα

Δεν πέρασα τα παρακάτω, γιατί το reverse geocoding τα δείχνει σε άλλη Περιφερειακή Ενότητα:

- Παραλία Ρούγας: Αιτωλοακαρνανία
- Μυρτάρι: Αιτωλοακαρνανία
- Αλωνάκι: Πρέβεζα
- Παντοκράτορας: Πρέβεζα
- Μονολίθι: Πρέβεζα
- Κυανή Ακτή: Πρέβεζα

Άρα αυτά δεν πρέπει να εμφανίζονται ως παραλίες Άρτας.

## Τι διορθώθηκε στον έλεγχο

Πρόσθεσα καθαρά γεωγραφικά όρια για `epirus-arta-mainland` στα audit scripts.

Αυτό σημαίνει ότι στο μέλλον το audit της Άρτας δεν θα βασίζεται σε τυχαίο auto-bbox από τις υπάρχουσες παραλίες και δεν θα τραβάει εύκολα παραλίες από Πρέβεζα ή Αιτωλοακαρνανία.

## Πηγές ελέγχου

- OpenStreetMap Overpass για beach candidates.
- Nominatim reverse geocoding για να δω σε ποια Περιφερειακή Ενότητα ανήκει κάθε candidate.

## Τρέχον command

```bash
npm run audit:beaches -- --island=arta --mode=deep --email-dry-run --refresh-external
```

Αποτέλεσμα:

```text
Beaches checked: 2
Issues: 0 (BLOCKER 0, HIGH 0, MEDIUM 0, LOW 0)
Gate status: pass for BLOCKER/HIGH issues
External coverage: 2 candidates from OpenStreetMap Overpass
```

## Ρίσκα που μένουν

- Η Άρτα έχει κυρίως ακτές Αμβρακικού/λιμνοθάλασσας, όχι κλασικό ανοιχτό πέλαγος. Αν αργότερα αποφασίσουμε ότι το BeachBuddy πρέπει να εμφανίζει μόνο “κανονικές” τουριστικές παραλίες ανοιχτής θάλασσας, η Άρτα θέλει product απόφαση.
- Δεν πρόσθεσα amenities ή χαρακτηριστικά με υψηλή βεβαιότητα, γιατί δεν υπήρξε αρκετή αξιόπιστη τοπική επιβεβαίωση μέσα από αυτόν τον έλεγχο.
