# Ηράκλειο Κρήτης - Batch 2 προσθήκες

Ημερομηνία: 2026-05-23

## Τι έγινε

Περάστηκε δεύτερος γύρος για το Ηράκλειο Κρήτης, με βάση το deep audit και το OpenStreetMap/Overpass cache.

Το dataset του Ηρακλείου πήγε από 28 σε 51 παραλίες.

Δεν πέρασα τυφλά όλα τα ευρήματα. Όσα ήταν διπλά sections της ίδιας παραλίας μπήκαν ως μία εγγραφή. Όσα έμοιαζαν με aliases μεταφέρθηκαν στο σωστό σημείο.

- Το `Σαραντάρι / Sarantari / Saradari` μεταφέρθηκε από τα aliases της εγγραφής `Λιμανάκια Χερσονήσου` στη νέα εγγραφή `Σαραντάρι`.

## Αποτέλεσμα μετά τον δεύτερο γύρο

Η εντολή:

```bash
npm run audit:beaches -- --island=heraklion --mode=deep --email-dry-run --refresh-external
```

έβγαλε:

- Παραλίες που ελέγχθηκαν: 51
- BLOCKER: 0
- HIGH: 0
- MEDIUM: 1
- LOW: 5
- Gate status: περνάει για blocker/high προβλήματα

Πριν τον δεύτερο γύρο υπήρχαν 28 audit issues. Μετά τον δεύτερο γύρο έμειναν 6.

## Τι έμεινε

Το μόνο MEDIUM που έμεινε είναι:

- `Ψιλή Άμμος`: το OSM έχει δεύτερο σημείο με ίδιο όνομα περίπου 5.6km από την υπάρχουσα εγγραφή. Αυτό δεν το άλλαξα αυτόματα, γιατί μπορεί να είναι δεύτερη παραλία με ίδιο όνομα και χρειάζεται καθαρή ονομασία διάκρισης πριν μπει στο dataset.

Τα LOW που έμειναν είναι κυρίως coordinate precision notes:

- `Keratokambos`: έχουμε μία εγγραφή για δύο κοντινά OSM sections.
- `Ληστής`: έχουμε μία εγγραφή για δύο κοντινά OSM sections.
- `Μάλια`: το OSM έχει `Aelos Beach` ως κοντινό section.

Αυτά δεν είναι blocking προβλήματα.

## Σημαντική σημείωση για confidence

Οι περισσότερες νέες εγγραφές του Batch 2 μπήκαν με `low` confidence για παροχές/χαρακτηριστικά, επειδή η πηγή τεκμηριώνει κυρίως την ύπαρξη και τη θέση της παραλίας, όχι πλήρως τις παροχές.

Δεν έγιναν δυνατές claims τύπου “σίγουρα οργανωμένη”, “σίγουρα ρηχή”, “ιδανική”, “προστατευμένη”.
