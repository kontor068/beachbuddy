## Τι δεν προστέθηκε επίτηδες

Δεν προστέθηκαν σαν κανονικές παραλίες:

- γυμνιστικά sections, γιατί το BeachBuddy δεν έχει ακόμα `nudism/FKK` πεδίο ή φίλτρο,
- beach-bar / ξενοδοχειακά / venue sections,
- μικρά υπο-τμήματα που είναι δίπλα σε υπάρχουσα παραλία και θα δημιουργούσαν διπλές κάρτες,
- δεύτερα σημεία της ίδιας μεγάλης παραλίας, π.χ. Νικήτη, Κομίτσα, Ουρανούπολη.

Αυτά πλέον εμφανίζονται ως LOW ενημερωτικά ευρήματα, όχι σαν προβλήματα που πρέπει να λυθούν.

## Audit rules που βελτιώθηκαν

- Προστέθηκαν όρια για `central-macedonia-halkidiki-mainland`, ώστε το audit να μην τραβάει λάθος ακτές από Σταυρό, Νέα Βρασνά, Καβάλα ή άλλες κοντινές περιοχές.
- Το OSM name handling βελτιώθηκε ώστε όταν το `name` είναι σκέτο `Παραλία`, να χρησιμοποιεί πιο συγκεκριμένο `name:el`, `name:en` ή `int_name` όπου υπάρχει.
- Προστέθηκε κανόνας για τη Χαλκιδική που υποβιβάζει nudist/venue/section candidates σε LOW αντί να τα ξαναβγάζει ως MEDIUM.

## Πηγές

- OpenStreetMap Overpass για named beach candidates.
- Nominatim reverse geocoding για επιβεβαίωση ότι τα ύποπτα σημεία ανήκουν στη Χαλκιδική.
- Visit Halkidiki spot checks για βασικές παραλίες όπως Toroni, Armenistis και Stratoni.

## Τρέχον αποτέλεσμα audit

Τελευταίο command:

```bash
npm run audit:beaches -- --island=halkidiki --mode=deep --email-dry-run
```

Αποτέλεσμα:

```text
Beaches checked: 124
Issues: 57 (BLOCKER 0, HIGH 0, MEDIUM 0, LOW 57)
Gate status: pass for BLOCKER/HIGH issues
Next action: No blocking or review-level issues found.
```

## Ρίσκα που μένουν

- Τα νέα records έχουν κυρίως low/medium confidence. Δεν τα θεωρούμε πλήρως verified.
- Τα amenities/access είναι συντηρητικές εκτιμήσεις, όχι πλήρης τοπική επιβεβαίωση.
- Η Χαλκιδική έχει πολλές μεγάλες παραλίες με πολλά sections. Για MVP κρατάμε την κάλυψη χρήσιμη, όχι υπερβολικά κομματιασμένη.
- Αν αργότερα θέλουμε nudist/FKK φίλτρο, τα deferred nudist sections πρέπει να ξαναπεραστούν με ξεχωριστό schema.
