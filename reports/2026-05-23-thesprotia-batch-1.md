## Τι διορθώθηκε στον έλεγχο

Προστέθηκαν καθαρά γεωγραφικά όρια για `epirus-thesprotia-mainland`.

Αυτό προστατεύει το audit από:

- παραλίες Πάργας/Πρέβεζας στα νότια,
- παραλίες Κέρκυρας δυτικά,
- λάθος auto-bbox που χάνει βόρεια Θεσπρωτία προς Σαγιάδα.

Επίσης σημειώθηκαν ως ήδη καλυμμένα sections:

- δεύτερο OSM section για Δρέπανο,
- δεύτερο OSM section για Μακρυγιάλι,
- δεύτερο OSM section για Πλαταριά.

Δεν δημιουργήθηκαν ξεχωριστές κάρτες για αυτά, γιατί θα έκαναν το app πιο μπερδεμένο χωρίς καθαρό όφελος για MVP.

## Τρέχον command

```bash
npm run audit:beaches -- --island=thesprotia --mode=deep --email-dry-run --refresh-external
```

Αποτέλεσμα:

```text
Beaches checked: 39
Issues: 3 (BLOCKER 0, HIGH 0, MEDIUM 0, LOW 3)
Gate status: pass for BLOCKER/HIGH issues
```

## Ρίσκα που μένουν

- Οι 3 νέες εγγραφές θέλουν τοπική επιβεβαίωση για παροχές, βάθος και ακριβή χαρακτήρα παραλίας.
- `Παραλία Κέρκυρα` μπορεί να λειτουργεί ως section της μεγάλης ζώνης Δρέπανο/Μακρυγιάλι. Την πέρασα επειδή είναι ξεχωριστό OSM beach candidate και ανήκει στη Θεσπρωτία, αλλά με low confidence.
- Το `Keramidi Beach` χρειάζεται επιβεβαιωμένο ελληνικό όνομα πριν ανέβει confidence.
